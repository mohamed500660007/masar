# Interactive Guide to OS Processes

In an Operating System, a **process** is a program in execution. While a program is a passive entity (like a file on disk), a process is an active entity with a program counter, registers, memory allocations, and system resources.

---

## 2. Interactive Process Life Cycle

As a process executes, it moves through different states. The CPU scheduler transitions processes between these states to optimize CPU utilization.

Step through the process simulation below to run a task state machine:

export const ProcessSimulator = () => {
  const [state, setState] = useState('NEW');
  const [log, setLog] = useState('Process is being initialized in storage.');
  const [history, setHistory] = useState(['NEW']);
  const transition = (newState, reason) => {
    setState(newState);
    setLog(reason);
    setHistory(prev => [...prev, newState]);
  };
  const reset = () => {
    setState('NEW');
    setLog('Simulation restarted. Process created.');
    setHistory(['NEW']);
  };
  const btnStyle = {
    padding: '8px 14px',
    background: '#6366F1',
    color: 'white',
    border: 'none',
    borderRadius: '6px',
    cursor: 'pointer',
    fontWeight: 600,
    fontSize: '0.85rem',
    transition: 'background 0.2s',
  };
  return (
    <div style={{
      border: '1px solid #1E293B',
      background: '#0F172A',
      padding: '1.5rem',
      borderRadius: '12px',
      marginTop: '1.5rem',
      marginBottom: '1.5rem'
    }}>
      <div style={{ display: 'flex', justifyContent: 'space-between', alignItems: 'center', marginBottom: '1.25rem' }}>
        <h4 style={{ color: '#F8FAFC', margin: 0, fontSize: '0.95rem' }}>Lifecycle State Machine</h4>
        <span style={{
          background: state === 'RUNNING' ? '#10B981' : state === 'WAITING' ? '#F59E0B' : state === 'READY' ? '#6366F1' : '#64748B',
          color: '#020617',
          padding: '4px 12px',
          borderRadius: '9999px',
          fontWeight: 800,
          fontSize: '0.75rem',
          letterSpacing: '1px'
        }}>{state}</span>
      </div>
      <div style={{
        background: '#020617',
        padding: '12px',
        borderRadius: '6px',
        border: '1px solid #1E293B',
        fontSize: '0.85rem',
        color: '#E2E8F0',
        marginBottom: '1.25rem',
        fontFamily: 'monospace'
      }}>
        ℹ️ {log}
      </div>
      <div style={{ display: 'flex', gap: '8px', flexWrap: 'wrap', marginBottom: '1.25rem' }}>
        {state === 'NEW' && (
          <button onClick={() => transition('READY', 'Admitted to the primary ready queue in RAM.')} style={btnStyle}>Admit to Ready</button>
        )}
        {state === 'READY' && (
          <button onClick={() => transition('RUNNING', 'CPU Scheduler Dispatch: allocated core execution time.')} style={btnStyle}>Dispatch to CPU</button>
        )}
        {state === 'RUNNING' && (
          <div style={{ display: 'flex', gap: '8px', flexWrap: 'wrap' }}>
            <button onClick={() => transition('READY', 'Interrupt: Timer slice expired (Preemption).')} style={{ ...btnStyle, background: '#EF4444' }}>Interrupt (Timeout)</button>
            <button onClick={() => transition('WAITING', 'I/O Request: blocked waiting for disk/network resource.')} style={{ ...btnStyle, background: '#F59E0B' }}>Request I/O</button>
            <button onClick={() => transition('TERMINATED', 'Exit: task execution finished successfully.')} style={{ ...btnStyle, background: '#10B981' }}>Finish / Exit</button>
          </div>
        )}
        {state === 'WAITING' && (
          <button onClick={() => transition('READY', 'I/O Event Completed: moved back to the ready queue.')} style={btnStyle}>I/O Completed</button>
        )}
        {state === 'TERMINATED' && (
          <button onClick={reset} style={{ ...btnStyle, background: '#475569' }}>Restart Simulator</button>
        )}
      </div>
      <div style={{ fontSize: '0.8rem', color: '#64748B' }}>
        <strong>State History Trace: </strong>
        {history.join(' ➔ ')}
      </div>
    </div>
  )
}

## 3. The Process Control Block (PCB)

For each process, the Operating System maintains a **Process Control Block (PCB)**. Think of this as the process's ID card or snapshot.

Use the interactive inspector below to view what the OS tracks inside a PCB:

## 4. Context Switching

When the CPU scheduler suspends execution of **Process A** and starts executing **Process B**, it triggers a **Context Switch**.

Click the simulator below to step through how the CPU transfers states:

export const ContextSwitcher = () => {
  const [step, setStep] = useState(0);
  const steps = [
    {
      action: "Process A is executing on the CPU core.",
      pcbA: "State: Running | PC: 0x04 | Registers: Active",
      pcbB: "State: Ready | PC: 0xA0 | Registers: Saved",
      cpu: "Executing Process A"
    },
    {
      action: "Interrupt occurs. CPU suspends Process A and saves its register values to PCB A.",
      pcbA: "State: Ready | PC: 0x05 | Registers: SAVED ✅",
      pcbB: "State: Ready | PC: 0xA0 | Registers: Saved",
      cpu: "Idle (Saving State A...)"
    },
    {
      action: "CPU loads the execution context (PC, register values) from PCB B.",
      pcbA: "State: Ready | PC: 0x05 | Registers: Saved",
      pcbB: "State: Running | PC: 0xA0 | Registers: LOADED 🔄",

      action: "Process B starts execution on the CPU core. Context switch complete.",
      pcbA: "State: Ready | PC: 0x05 | Registers: Saved",

  return (
    <div style={{
      border: '1px solid #1E293B',

      marginTop: '1.5rem',
      marginBottom: '1.5rem'
    }}>

        fontSize: '0.85rem',
        color: '#E2E8F0',
        marginBottom: '1rem',
        borderLeft: '4px solid #6366F1'
      }}>
        <strong>Step {step + 1}:</strong> {steps[step].action}
      </div>
      <div style={{ display: 'grid', gridTemplateColumns: '1fr 1fr 1fr', gap: '8px', marginBottom: '1.25rem' }}>
        <div style={{ background: '#1E293B', padding: '10px', borderRadius: '6px', fontSize: '0.8rem' }}>

        </div>
        <div style={{
          background: '#020617',
          padding: '10px',
          borderRadius: '6px',
          fontSize: '0.8rem',
          display: 'flex',
          flexDirection: 'column',
          alignItems: 'center',
          justifyContent: 'center',
          border: '1px dashed #334155'
        }}>
          <strong style={{ color: '#6366F1', display: 'block', marginBottom: '4px' }}>CPU</strong>
          <span style={{ color: '#F8FAFC', fontWeight: 'bold' }}>{steps[step].cpu}</span>
        </div>
        <div style={{ background: '#1E293B', padding: '10px', borderRadius: '6px', fontSize: '0.8rem' }}>
          <strong style={{ color: '#10B981', display: 'block', marginBottom: '4px' }}>PCB B</strong>
          <span style={{ color: '#CBD5E1', fontFamily: 'monospace' }}>{steps[step].pcbB}</span>
        </div>
      </div>
      <div style={{ display: 'flex', gap: '8px' }}>
        <button 
          onClick={() => setStep(prev => Math.max(0, prev - 1))}
          disabled={step === 0}
          style={{
            padding: '6px 12px',
            background: step === 0 ? '#475569' : '#1E293B',
            color: '#F8FAFC',
            border: 'none',
            borderRadius: '4px',
            cursor: step === 0 ? 'not-allowed' : 'pointer',
            fontSize: '0.8rem',
            fontWeight: 600
          }}
        >
          Previous Step
        </button>
        <button 
          onClick={() => setStep(prev => Math.min(steps.length - 1, prev + 1))}
          disabled={step === steps.length - 1}
          style={{
            padding: '6px 12px',
            background: step === steps.length - 1 ? '#475569' : '#6366F1',
            color: 'white',
            border: 'none',
            borderRadius: '4px',
            cursor: step === steps.length - 1 ? 'not-allowed' : 'pointer',
            fontSize: '0.8rem',
            fontWeight: 600
          }}
        >
          Next Step
        </button>
        {step === steps.length - 1 && (
          <button 
            onClick={() => setStep(0)}
            style={{
              padding: '6px 12px',
              background: '#475569',
              color: '#F8FAFC',
              border: 'none',
              borderRadius: '4px',
              cursor: 'pointer',
              fontSize: '0.8rem',
              fontWeight: 600
            }}
          >
            Reset
          </button>
        )}
      </div>
    </div>
  )
}

> [!NOTE]
> **Performance Overhead:** Context switching is pure overhead. The system does no useful computation while saving and restoring registers and memory contexts, meaning highly frequent switching (thrashing) degrades performance.
